입력하신 이메일로 등록된 ID와 Password를 자동으로 통보해 드립니다.
3~5분 경과후 이메일을 수신해 보세요.

## 서버데몬을 이용한 로그인으로 훨씬 더 강화된 보안도구를 제공합니다.

** 설치방법 **

1. 먼저 압축을 풀어서 다음과 같은 파일이 있는 지 확인합니다.

  • memberinfo (email.txt, footer.txt, header.txt)
  • memberonly (.htaccess)
  • pass (디렉토리만 있고 파일 없음)
  • acctman.cgi
  • amadmin.cgi
  • config.pl
  • index.html
  • amform.html
  • finder.html

2. 먼저 펄 경로를 확인 합니다.
텔넷 접속후 "which perl", 각 cgi 프로그램의 첫줄 #! /usr/bin/perl 을 사용자의 계정에 맞게 수정합니다.

3. config.pl 파일을 열어서 환경 변수를 수정합니다.
자세한 설명은 파일내에 주석형태로 포함되어 있습니다. 대개의 경우 수정할 필요 없을 겁니다.

4. 서버에 업로드하고 퍼미션을 변경합니다.
"cd AMLite" (AMLite 디렉토리에 업로드했다고 가정했을 경우)
"chmod 777 *"
"chmod 755 *.cgi"
"chmod 755 *.pl"
"cd memberonly"
"pwd" 앞의 명령어를 실행한 후 나온 결과물은 메모해 놓습니다.
제 홈계정의 경우는 /home1/webweaver/htdocs/am/memberonly 라고 나왔습니다.

5. 다시 로컬 컴퓨터에 있는 memberonly 디렉토리로 이동합니다.
".htaccess" 파일을 메모장에서 열어봅니다.

AuthType Basic
AuthGroupFile /dev/null
AuthName "회원 ID 및 패스워드를 입력해 주세요!"
AuthUserFile /home1/webweaver/htdocs/am/memberonly/.htpasswd

<limit GET POST>
require valid-user
</limit>

4번에서 메모했던 경로를 빨간색으로 된 부분에 넣고 저장한후 다시 서버에 업로드합니다.

6. 일단 일단계가 완료 되었습니다.
브라우저로 테스트 해보세요.
"http://yourdomain.com/AMLite/"

7. 임의로 한명의 회원을 가입시켜 보십시오.
(이때는 테스트 목적이므로 e-mail 부분에 관리자의 e-mail을 입력하세요.)
그리고, 관리자 도구로 들어가서 회원 승인을 해보세요.
귀하에게 가입신청확인, 승인 등을 확인하는 메일이 도착했습니까?
메일이 오지않았다면 "config.pl" 파일을 다시 한번 확인해 보세요.

"http://yourdomain.com/AMLite/memberonly/" 를 브라우저에서 불러 보세요.
방금 가입한 아디와 패스로 로그인 해보세요. 페이지가 나타나면 성공.

8. 접근을 제한하려는 페이지를 "memberonly" 디렉토리에 넣으시면 됩니다.

9. "amform.html" 파일은 회원가입폼이니 고쳐사용하세요. "finder.html" 파일은 아디 분실신고를 하는 폼입니다. 원하는 페이지에 붙여서 사용하시면 됩니다.

10. 문의사항은 http://www.webweaver.pe.kr 질답 게시판을 이용해 주세요.

* 게시판과의 연동을 문의하시는 분이 많으신데요.
흔히 게시판은 인덱스 파일(글번호, 작성자이름, 날짜, 메일주소..)과 데이터 파일(실제 글이 저장되는 텍스트 파일)로 나누어져 있습니다. 이경우 데이터파일 저장경로를 memberonly 디렉토리로 하면 되겠죠?
